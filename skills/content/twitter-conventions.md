# Twitter/X Conventions

Platform rules and norms for posting on Twitter/X as part of Mellea developer relations.

## Character Limits

- **Single tweet**: 280 characters
- **Thread tweet**: 25,000 characters per tweet (but keep them under ~280 for readability)
- **URLs**: Always consume exactly 23 characters regardless of actual URL length
- **Images**: Add no characters but significantly increase engagement

When to use a thread vs a single tweet:
- Single tweet: one insight, one demo, one announcement hook
- Thread: step-by-step walkthrough, before/after comparison, or a point that needs 3+ supporting examples

## Code in Tweets

- Max 3-5 lines of code in a single tweet before it becomes unreadable
- For longer snippets: take a screenshot with large font (e.g., Carbon, ray.so) and attach as image
- Or link to a Gist, GitHub file, or blog post
- Highlight the key line, not the entire file

## Hashtags

- **Generally skip them** unless targeting a specific community (e.g., `#PyTorch` for ML audience)
- If using hashtags: 1-2 maximum, placed at the end, only if they're genuinely relevant
- Hashtags that don't match the content look spammy
- Relevant hashtags for Mellea: `#Python`, `#LLM`, `#OpenSource`, `#MachineLearning`

## Link Behavior

- Links go at the end of the tweet
- A tweet with a link gets slightly less reach — front-load the value before the link
- To maximize reach: post the insight without a link first, add link in a reply
- Use link previews to your advantage: make sure the page title/OG image are good

## Engagement Patterns

- Questions at the end of tweets get replies
- "What do you use for X?" outperforms "We built X for you"
- Showing code that solves a real problem outperforms describing the solution in prose
- Tag relevant accounts sparingly (max 1-2), only when they'd genuinely want to see it

## Mention Etiquette

- Tag someone when they contributed directly, not to boost reach
- Don't tag people who didn't ask to be tagged in your content
- Tagging the official account (`@IBM`, `@IBMResearch`) is fine for IBM-voice posts
